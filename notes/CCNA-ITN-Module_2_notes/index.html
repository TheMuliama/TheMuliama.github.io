2.1 Cisco IOS Access

Operating systems

An Operating system (OS) has ==two portions: the Shell and the Kernel==. The ==Shell interact with the user==. The ==Kernel interact directly with the Hardware of the device==.

It's possible for the user to interact with a ==Command Line Interface (CLI)== or a ==Graphical User Interface (GUI)==.

GUI is more user-friendly than CLI. ==CLI can let you do things impossible to do through GUI==. GUI can fail, crash or simply not work. CLI is far more stable but require to know the commands you want to use.

Network OS use a CLI because it's important for it to be stable and to act as specified.

Access Methods

All device should be configured and secured. In order to do that there is multiple way to access a network device.

==Out-of-band access==: ==access via a special port== that make device accessible even ==with no network connection==.

Console Physical port to access Cisco Device. Out-of-band access useful for first configuration. To access device you need a computer with a terminal emulation software and a console cable. Secure Shell (SSH) In-band acess and recommended method for remonte CLI connection. Require active networking services. Provide secure encrypted protection. Telnet Like SSH but without secure encrypted protection. Should only be used in lab environnement.

2.2 IOS Navigation

Primary Command Modes

User Exec mode:

  • Allow access to a limited number of basic monitoring commands
  • Often called "View-only" mode
  • Default device prompt:
Switch> 
Router>

Privileged EXEC mode:

  • Allow access to all command and features
  • Allow monitoring command, execute configuration and management commands
  • Default device prompt:
Switch#
Router#

Configuration Mode and Subconfiguration Modes

Global configuration mode is the mode to configure the device. From this mode we can access other specific config mode.

Two common subconfig mode are:

  • Line configuration mode - To configure console, SSH, Telnet, or AUX access.
  • Interface Configuration Mode - To configure a switch port or router network interface.

It's possible to identify which mode we are in with the command line prompt. A list of possible prompt for a Switch:

  • Switch(config)# : global config mode
  • Switch(config-line)# : Line configuration mode
  • Switch(config-if)# : Interface Configuration Mode

Navigate Between IOS Modes

enable: move from user EXEC mode to privileged EXEC mode disable: move from privileged EXEC mode to user EXEC mode

configure terminal: move in global config mode exit: to go back a configuration mode

line {management type line} {number}: to enter a line sub-configuration mode interface {management type interface} {number}: to enter a interface sub-configuration mode

end: to go from any subconfig mode to privileged EXEC mode

IOS Command Syntax Check

A command sometimes require keywords or arguments. To know what to we need to refer to the syntax.

Syntax give the format that must be used with a command.

Convention Description
boldface Indicate command and keywords that you enter literally as shown.
italics Indicate arguments for which you enter values.
[x] Optional element (keywords or arguments).
{x} Required element (keywords or arguments).
[x {y | z}] Braces and vertical lines within square braquets indicate a required choice within an optional element. Spaces are used to clearly identify parts of the command.
--------------------------------
The following examples demonstrate conventions used to document and use IOS commands:
  • ping ip-address - The command is ping and the user-defined argument is the ip-address of the destination device. For example, ping 10.10.10.5.
  • traceroute ip-address - The command is traceroute and the user-defined argument is the ip-address of the destination device. For example, traceroute 192.168.254.254.

IOS help Features

The IOS has two forms of help available: context-sensitive help and command syntax check.

Context-sensitive help enables you to quickly find answers to these questions:

  • Which commands are available in each command mode?
  • Which commands start with specific characters or group of characters?
  • Which arguments and keywords are available to particular commands?

To access context-sensitive help, simply enter a question mark, ?, at the CLI.

Command syntax check verifies that a valid command was entered by the user. If the interpreter cannot understand the command being entered, it will provide feedback describing what is wrong with the command.
